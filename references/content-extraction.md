# Content Extraction — Transcripts & Long-form Text

Lazy-loaded in Phase 3 when a content URL (YouTube, podcast, blog, talk,
substack/medium, x thread) is encountered. Phase 3 rule: **when a content
surface is found, extract the text on the spot** — don't just record the
URL. Pull it while the platform context is hot.

All extracted text → `./osint-<slug>/content/<platform>-<id>.md`. One
file per piece of content. See §"Storage layout" for the full shape.

---

## YouTube

### Primary — Apify transcript actor

Use the actor catalogued in `tools.md` §"YouTube — transcript".

```
apify call <transcript-actor> --input '{"videoUrls":["https://youtube.com/watch?v=<id>"],"language":"en"}' --silent > ./osint-<slug>/scratch/yt-<id>.json
```

Output: a JSON envelope with `result.transcript` (array of `{start, dur,
text}`) or `result.text` (concatenated string). Auto-captions are flagged
`isAutoGenerated: true` — keep this flag in the file header so Phase 5
(psychoprofile) can weight accordingly.

Write to `./osint-<slug>/content/youtube-<videoId>.md`.

### Fallback — yt-dlp

Only if the Apify actor returns an empty transcript (captions disabled).
`yt-dlp` is blocked for downloading video, but the caption-only path
still works on most uploads.

```
yt-dlp --write-auto-sub --sub-lang en --skip-download \
  -o './osint-<slug>/scratch/yt-<id>.%(ext)s' 'https://youtube.com/watch?v=<id>'
```

Output: `./osint-<slug>/scratch/yt-<id>.en.vtt`. Strip timestamps, cue ids, and consecutive
duplicate lines (auto-captions repeat across cue boundaries):

```
sed -E '/^[0-9]+$/d; /-->/d; /^WEBVTT/d; /^$/d' ./osint-<slug>/scratch/yt-<id>.en.vtt \
  | awk '!seen[$0]++' > ./osint-<slug>/scratch/yt-<id>.txt
```

If `yt-dlp` also fails (HTTP 429 or `unable to extract`), fall back to
**whisper** on the audio: `yt-dlp -x --audio-format mp3 -o './osint-<slug>/scratch/yt-<id>.mp3' <url>`
then run the whisper one-liner from §"Podcasts". Slow and costs more —
mark the dossier gap before launching it.

## Podcasts

### Step 1 — Find the RSS feed

- Apple Podcasts: `jina read https://podcasts.apple.com/.../id<showId>`
  then grep for `feedUrl` in the page source.
- Spotify: feed is hidden — use the show's own website or a public RSS
  resolver.
- Direct feed: `jina read <feed-url>` returns parseable XML/markdown.

### Step 2 — Extract the mp3 URL

From the RSS XML, the episode's `<enclosure url="..." type="audio/mpeg"/>`
attribute is the mp3. Episodes are listed newest-first.

### Step 3 — Transcribe

Two options. Pick by latency budget.

> **Carve-out:** The mp3 fetch below uses `curl` — an exception to the
> typed-CLI rule. No typed CLI in the toolchain fetches binary audio
> cleanly today. The downloaded file is consumed only by `whisper` /
> `whisper-cpp` for transcription; the binary content is never inhaled
> into agent context — only its plaintext transcript is. The scratch
> file lives under `./osint-<slug>/scratch/` and is deletable post-run.

**whisper.cpp (local, ~real-time on M-series, no API cost):**

```
curl -sL -o ./osint-<slug>/scratch/ep.mp3 '<mp3-url>' && \
  whisper-cpp -m ~/models/ggml-base.en.bin -f ./osint-<slug>/scratch/ep.mp3 -otxt -of ./osint-<slug>/scratch/ep
```

Cost: $0. Latency: ~1× audio duration on `base.en`, ~0.3× on metal-
accelerated `large-v3`.

**OpenAI Whisper API (faster on long episodes, has cost):**

```
curl -sL -o ./osint-<slug>/scratch/ep.mp3 '<mp3-url>' && \
  openai api audio.transcriptions.create -m whisper-1 -f ./osint-<slug>/scratch/ep.mp3 \
  --response-format text > ./osint-<slug>/scratch/ep.txt
```

Cost: ~$0.006/min of audio. A 60-min episode = $0.36. **Always check the
dossier budget before kicking off** — multi-episode podcast runs blow
past the $0.50 default fast.

Write to `./osint-<slug>/content/podcast-<episode-slug>.md`.

## Blogs / long-form articles

### Primary — jina

```
jina read 'https://example.com/post' > ./osint-<slug>/scratch/post.json
```

Returns clean markdown of the article body in `result.content`. Strips
nav, ads, footer.

### Fallback — brightdata

```
brightdata scrape 'https://example.com/post' > ./osint-<slug>/scratch/post.json
```

Use when `jina` returns empty (some sites detect Jina's reader fetcher),
or when the page renders content via JS that Jina misses. Brightdata
uses a real browser.

### Paywalled articles

**Operator-decision-point.** Do not bypass paywalls without explicit
authorisation. Options:

- Operator has a personal subscription → operator-side fetch, paste-back.
  Treat the resulting content as Grade B (single source).
- Article is public-after-N-days → record the URL, mark the gap with the
  date when it should become accessible, move on.
- Article is paywalled with a free abstract → extract the abstract via
  `jina read`, label the rest as `[paywalled — abstract only]`.

Never recommend or implement bypass workarounds (paywall-remover sites,
login-bypass extensions). The skill's brief is graded public-record
dossiers — paywalled content is not public-record.

Write to `./osint-<slug>/content/blog-<host>-<slug>.md`.

## Conference talks

Talks are scattered across YouTube channels, Vimeo, custom players, and
Notion-embedded videos. Recognition heuristics:

- **YouTube-hosted** (most conferences): use the YouTube path above.
- **Vimeo:** URL pattern `vimeo.com/<id>` or `player.vimeo.com/video/<id>`.
  `yt-dlp` works on most public Vimeo videos. Captions when present:
  `yt-dlp --write-auto-sub --sub-lang en --skip-download <url>`. No
  captions → whisper on audio.
- **Custom players** (Brightcove, JWPlayer, conference-specific):
  recognise by the absence of any of the above patterns and the presence
  of `<video>` tags or `.m3u8` URLs in the page source. `yt-dlp` handles
  many; failing that, `brightdata scrape` of the page often surfaces a
  direct mp4 URL in the rendered HTML.
- **Notion-embedded videos:** the iframe usually points to one of the
  above. `jina read` the Notion page, find the iframe `src`, follow it.

Talks are the highest-signal content surface for psychoprofile when
available — long-form, prepared, on-record. Prioritise these over
short-form social posts when both exist.

Write to `./osint-<slug>/content/talk-<conference>-<year>-<slug>.md`.

## Substack / Medium

### Primary — RSS

Both expose full-text RSS:

- Substack: `https://<publication>.substack.com/feed`
- Medium (publication): `https://medium.com/feed/<publication>`
- Medium (user): `https://medium.com/feed/@<user>`

`jina read <feed-url>` returns the XML as readable markdown with full
article bodies inlined. Per item, write a separate file under
`./osint-<slug>/content/substack-<post-slug>.md` (or `medium-<post-slug>.md`).

### Fallback — jina on the article URL

If RSS is paywalled (some Substack publications gate full text behind
paid subs, with only a preview in the feed):

```
jina read 'https://<publication>.substack.com/p/<post-slug>'
```

If still partial → mark `[paywalled — preview only]` per the blog
paywall rule above.

## X / Twitter threads

X's auth-walling has made direct thread extraction unreliable. Working
path:

### Primary — Nitter via jina

Nitter is a privacy-respecting front-end for X. Multiple public mirrors
exist; they go up and down individually but at least one is usually live.
Known mirrors: `nitter.net`, `nitter.privacydev.net`, `nitter.poast.org`.
Substitute `<mirror>` with whichever is currently responsive — `WebFetch
https://<mirror>/` 200 OK = up.

```
jina read 'https://<mirror>/<handle>/status/<tweet-id>'
```

Output: the rendered thread as markdown, including replies from the same
author (true thread continuation).

**Reality check:** Nitter is intermittent. Mirrors get rate-limited by
X, go offline for hours or days, then come back. If the first mirror
returns empty or 5xx, try one more. If two fail, switch to fallback.

### Fallback — brightdata on x.com

```
brightdata scrape 'https://x.com/<handle>/status/<tweet-id>'
```

Slower, costs more, but works against the live x.com page when Nitter
is down.

### Last resort — apify tweet actor

Per `tools.md` §"X — tweets", with the warning that historical
pagination caps make deep threads partial.

Write to `./osint-<slug>/content/x-thread-<tweet-id>.md`.

## Storage layout

```
./osint-<slug>/
  dossier.md
  seed.json
  seed-summary.md
  phase-2-raw.md          # gated, may not exist
  platform-*.md           # one per platform (Phase 3)
  content/
    youtube-<videoId>.md
    podcast-<slug>.md
    blog-<host>-<slug>.md
    talk-<conf>-<year>-<slug>.md
    substack-<slug>.md
    medium-<slug>.md
    x-thread-<tweetId>.md
```

One file per content piece. File header shape (all platforms, fields
omitted when not applicable):

```
# <Title>
- Source: <url>
- Author / Speaker / Channel: <name>
- Date: <YYYY-MM-DD>
- Duration: <mm:ss or hh:mm>
- Extraction tool: <tool + variant>
- Quality flag: auto-generated | paywalled-partial | (none)

---

<extracted text>
```

The dossier cites these by relative path so review can read the full
transcript without re-fetching.

## Cost & latency rule of thumb

| Surface | Cost | Latency |
|---|---|---|
| YouTube transcript (apify) | ~$0.0005/video | seconds |
| YouTube transcript (yt-dlp + vtt) | $0 | seconds |
| YouTube whisper fallback | ~$0.006/audio-min (api) or $0 (local) | minutes |
| Podcast whisper.cpp local | $0 | ~1× audio duration |
| Podcast whisper API | ~$0.006/min | ~0.1× audio duration |
| Blog (jina) | ~$0.001/page | seconds |
| Blog (brightdata) | ~$0.005/page | ~10s |
| Conference talk (yt-dlp + vtt) | $0 | seconds |
| Substack / Medium (jina) | ~$0.001/feed or page | seconds |
| X thread (jina via nitter) | ~$0.001/thread | seconds (when up) |
| X thread (brightdata) | ~$0.005/thread | ~10s |

Default to the cheap path. Escalate only when the cheap path returns
empty content. Track all spend in the dossier audit log.
